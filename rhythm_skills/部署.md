# 部署

将当前改动构建并发布到 GitHub Pages。

## 步骤

1. 检查工作区状态：
   ```bash
   git status
   git diff --stat
   ```

2. 在本地执行构建，确认无报错：
   ```bash
   cd /Users/koishi810/Desktop/rhyme-dict-web
   npm run build
   ```

3. 构建成功后，提交所有改动：
   - 让用户提供本次改动的简短描述
   - 执行 `git add -A && git commit -m "{描述}"`

4. 推送到 GitHub：
   ```bash
   git push origin main
   ```

5. 推送后 GitHub Actions 会自动触发构建和部署，告知用户：
   - 进度页面：https://github.com/koishi810/rhyme-dict-web/actions
   - 部署完成后访问：https://koishi810.github.io/rhyme-dict-web

## 常见问题
- 构建报错 `Type error`：先修复 TypeScript 错误再推送
- Actions 失败：查看 Actions 日志，通常是依赖安装问题
- 页面样式丢失：确认 `next.config.ts` 中 `basePath: '/rhyme-dict-web'` 存在
